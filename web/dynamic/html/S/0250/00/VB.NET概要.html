VB.NET

VB.NETとは「Visual Basic .NET」の略で、マイクロソフト社が開発しているプログラミング言語です。
1991年に開発されて以来、多くのユーザを持ち、現在も大きなシェアを占めています。
また、Excelなどのマイクロソフトオフィスソフトで動作するVBAにも拡張されているため、VB.NETをマスターすればExcelなどのマクロも作成できるようになります。

一般パソコン向けOSのシェアナンバーワンであるWindows上での動作を強みとしますが、.NETフレームワークによってOSに関わらず動作させることが可能です。
近年の.NETフレームワークのさらなる普及を受けて人気もさらに高まってきています。

是非、VB.NETをマスターしましょう♪

VB.NETの歴史

Visual Basicの歴史を簡単に学びましょう♪
以下の4つの言語の過程を経て、現在に至ります。
  1. Quick Basic
  2. Visual Basic
  3. Visual Basic.NET
  4. Visual Basic(.NET)

Quick Basic

1985年に発売されたMS-DOM上で動作するプログラミング言語です。
Visual Basicとは系統が若干異なりますが、Visual Basicの前身であったと言えます。

Visual Basic

1991年にQuick Basicを大幅に改良して開発されました。
主にWindows上で動作することを想定し、またフォームを用いたデスクトップアプリケーション開発を可能にしました。
バージョンは「6」まであります。

Visual Basic.NET

Visual Basicのバージョン「7」からは言語使用が大幅に変更されて、マイクロソフト社が展開している.NETフレームワークに対応したことによって名称がVisual Basic.NETとなりました。

Visual Basic(.NET)

2005年からは(.NET)がつかなくなりました。
言語使用はVisual Basic.NETと同じです。
多くのユーザはVisual Basic.NETという名称をまだ使用しています。
マイクロソフト社の製品でも未だに「.NET」という名前を付けてVisual Basicを紹介していることが多いため、ここでもVB.NETと呼びことにします。

コンパイルとビルド

コンピュータは我々が書いた英語みたいなコードを直接実行することはできません。
コンピュータが実行することができるのは「0」と「1」だけで書かれたバイナリデータと呼ばれるものだけです。
そのため、我々が書いたコード(ソースコード)をコンピュータが実行できるように変換する必要があります。
これを「コンパイル」ないしは「ビルド」と呼びます。

コンパイル

プログラミングでは、英語っぽいコード(ソースコード)を入力しますがコンピュータはこのコードを直接実行することはできません。

コンピュータが我々が書いたソースコードを実行するためには一度ソースコードをコンピュータが実行できるコード(実行可能プログラム)へ変換する必要があります。

この作業を「コンパイル」と呼びます。

ビルド

コンパイルの進化バージョンです。
入力されたコードをチェックして、さらには実行時に必要なライブラリを結合させて実行可能なプログラムを作成することを言います。
ここでは、コンパイルとビルドを厳密に区別することはしませんが、これ以降にビルドと言ったら「コンパイル + α」の作業だと捉えてください。

環境構築

では、早速VB.NETプログラミングをする環境を整えましょう♪
プログラミングは主に以下の4つの工程を経ます。
  • コーディング(実際にコードを書く)
  • コンパイル
  • 実行
  • デバグ(不具合の原因の特定・修正)
これをALL IN ONEで実行する環境をIDEと言います。
ここではIDEとしてマイクロソフト社が開発しているVisual Studioを使用しましょう♪
個人であれば無料で使用できます。
以下のリンクへ飛んで、コミュニティを選択してダウンロードして下さい。 visual studio ダウンロードが完了したら、起動してください。
インストールの画面が表示されます。
ここでは、VB.NETの利用を想定するため、「.NETデスクトップ開発」を選択してください。
僕はそれ以外もインストールしていますが、無視してください。

すぐにユニバーサルwindowsプラットフォーム開発も使うことになると思うので、これもインストールすることをオススメします。

インストールが完了したら、プロジェクトを作成します。
プロジェクトとは複数のプログラムを包括する単位です。
「ファイル > 新規作成 > プロジェクト」からwindowsフォームアプリの作成を選択してください。
プロジェクト名は自由に設定してください。 プロジェクトの作成が完了すると、フォームが表示されます。
実際にコードを実行する場合には三角の緑色ボタンを押します。 コードを書いていないため、フォームのみ表示されます。

visual studioの使い方

visual studioの画面は大きく以下の4つの部分からなります。
  • コードウィンドウ
  • コンソールウィンドウ
  • ソリューションエクスプローラー
  • プロパティウィンドウ

コードウィンドウ

実際にコードを書いたり、フォームのレイアウトを操作したりする部分です。
実際にコードを各画面をコード編集画面、フォームを編集する画面をデザイン画面と呼びます。

コンソールウィンドウ

プログラムの出力内容を表示する部分です。
また、プログラムのコンパイラエラーが発生した場合は、その詳細を表示します。
主にデバグ目的で使用します。

ソリューションエクスプローラー

先ほどプログラムはプロジェクトという単位で管理すると言いましたが、プロジェクト内のファイルを管理する部分です。
最初のうちはほとんど使用しません。
