コンテナ

既に簡単に説明しましたが、コンテナとは論理的に区切られて構成されたアプリケーションの実行環境です。
ここではDockerを使用してコンテナを操作する方法を学びましょう♪

コンテナ操作

コンテナに対する処理は大きく種類があります。
  • start
  • stop
  • create
  • rm
  • run
  • exec
  • ls
  • ps
これらについてひとつずつ学びましょう♪
原則としてコマンドは以下のような構成をとります。 docker container サブコマンド オプション 対象 しかしながらcontainerは省略されて、dockerの後にサブコマンドを書くことが一般的です。 docker サブコマンド オプション 対象

start

指定したコンテナを開始します。
以下のように書きます。 docker container start コンテナ名 コンテナの実行には当然ですが、コンテナが必要です。

実行する前にcreateコマンドでコンテナを作成する必要があります。

stop

指定したコンテナを停止します。
以下のように書きます。 docker container stop コンテナ名

create

Dockerイメージからコンテナを作成します。
以下のように書きます。 docker create -オプション イメージ名 指定するイメージ名は事前にダウンロードする必要があります。
Dockerイメージの操作については後ほど説明します。

一般的にはDockerイメージのダウンロード・コンテナの作成・コンテナの実行をまとめて行ってくれるrunコマンドを使用します。

rm

指定したコンテナを削除します。
以下のように書きます。 docker container rm コンテナ名

run

Dockerイメージのダウンロード・コンテナの作成・コンテナの実行をまとめたものです。
この方法が一番簡単ですので、こちらを使用してイメージのダウンロード・コンテナの作成・コンテナの実行をしてみましょう♪

使用するアプリケーションはApacheです。
Apacheとは最も有名なWebサーバで、オープンソースで開発が進められています。

Apacheはhttpdという名前で登録されています。 docker container run --name test -d httpd 以下のコマンドで現在実行されているコンテナ一覧が取得できます。 docker container ps 以下のように出力されると思います。
runコマンドで使用されるオプションについて簡単に説明します。
--name コンテナの名前を指定します。
-d コンテナをバックグラウンドで実行します。
-i コンテナをキーボードで操作可能にします。
-t 特殊キーを使用可能にします。
-e キー=バリューの形で指定し、当該コンテナ内で有効な環境変数を登録します。
-p ポート番号を指定します。
-v 指定したボリュームをマウントします。
--net コンテナをネットワークに接続します。

exec

実行中のコンテナ内で、新しいコマンドを実行します。

ps

現在稼働中のコンテナ一覧を表示します。
「-a」オプションを付けることで停止しているコンテナも含めた全てのコンテナ一覧を出力します。 docker ps psコマンドは以下のコマンドの略称です。 docker container ls
