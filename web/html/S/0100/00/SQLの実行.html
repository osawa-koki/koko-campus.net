SQLの種類

SQL(Structed Query Language)は以下の3つに分類できます。
  • データ定義言語
  • データ制御言語
  • データ操作言語

データ定義言語

「Data Definition Language」で、「DDL」と略されます。
データベースのデータ構造を定義します。

表を作成する「CREATE」や表を削除する「DROP」が該当します。

データ制御言語

「Data Control Language」で、「DCL」と略されます。
データベースに対するアクセス制御を定義します。

権限を与える「GRANT」や権限を剥奪する「REVOKE」が該当します。

データ操作言語

「Data Manipulation Language」で、「DML」と略されます。
データの検索・更新・削除・追加を行うための言語です。

ここでは、このデータ操作言語について説明しますね♪

データ操作の種類

上で説明した通り、データ操作は以下の4つに分類されます。
  • 検索
  • 更新
  • 削除
  • 追加
因みに、データに対する処理4つを総称して「CRUD」と呼びます。
「CRUD」は以下の4つの処理の頭文字です。
  • C: Create
  • R: Read
  • U: Update
  • D: Delete

検索

検索文の基本構造は「SELECT 列名 FROM テーブル名 WHERE 条件;」です。 例えば以下のデータを保存しているデータベースの「generation3」表からtype1が「ほのお」である「name」を抜き出すSQLはこうなります。 SELECT name FROM generation3 WHERE type1 = 'ほのお';

全てのカラムを検索する際には「SELECT * FROM ～～～」のように、「*(アスタリスク)」を使用します。

このように更新用のSQLは「UPDATE テーブル名 SET 列名 = 値 WHERE 条件;」で作成します。 SETの内容が複数存在する場合は「SET type1 = 'エスパー', type2 = 'フェアリー'」のように、「,(カンマ)」で区切って表します。

WHERE句で条件を設定しないと全部のデータが変更されることに注意してください。
