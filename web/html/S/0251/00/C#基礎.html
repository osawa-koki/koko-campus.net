C#コードのひな型

using System; class ConsoleApp { static int Main() { // プログラムの開始点 // ここにプログラムコードを書きます！ return 0; } } コンソールアプリを開発する際には、最低1つのクラスとメソッドが必要で、メソッドの名前は「Main」とする必要があります。

クラス名は任意でOK!です。

「Main」と名付けられたメソッドがプログラムへのエントリーポイントとなります。

コメントアウト

プログラムコード中に可読性・保守性の観点からメモ書きをすること場合があります。
これをコメントアウトと呼びますが、コメントアウトの記述方法として以下の2通りがあります。
  • // コメント文
  • /* コメント文 */
// コメント文 /* コメント文 */
その行の「//」以降をコメント化します。 「/*」と「*/」で囲まれた部分全体(改行を含む)をコメント化します。
var x = 0; // コメントアウトです。 var y = 10; /* ここは コメントアウト です♪ */ var y = 25;

命令文

プログラムの処理は命令の集合からなります。
各々の命令のことを「命令文」といいますが、各命令文の終わりには「;(セミコロン)」を書きます。 命令文; 命令文; 命令文; この説明だけだと命令文がなんだか分からないですよね、、、
具体的には、変数の宣言・代入、メソッドの実行、インスタンスの生成などが該当しますが、ここでは処理の最小単位と認識しておいてください。

変数

変数とはプログラム実行時に使用する一時的なデータを格納する箱を言います。
あくまでも、プログラム実行のために使用されるもので、プログラムの実行が終了すると自動で解放されます。

データの永続化のためには、ファイルに書き込んだり、データベースに保存する必要があります。

変数の使用に関しては以下の3つの処理を学ぶ必要があります。
  1. 変数の宣言
  2. 変数の代入
  3. 変数の参照

変数の宣言

これから「〇〇〇」という名前でこんな種類のデータを扱います♪とプログラムに伝える作業です。
以下のように書きます。 データ型 変数名 データ型については次のページで詳しく説明しますが、ここでは「データの種類」のことだと思ってください。

代表的なデータ型には以下のものがあります。
int 整数
double 小数
string 文字列
boolean 真偽値
例えば、整数を格納する変数「num」を宣言する場合には以下のように書きます。 int num; これで、「num」という変数に整数型のデータを格納できるようになりました。

変数の代入

宣言した変数に実際にデータを代入するには、「=」を使用します。
例えば、先ほど宣言した「num」変数に「10」を代入するには以下のように書きます。 num = 10; 「=」というと、「等価」を意味しそうですが、プログラミングの世界では「代入」を表すことが多いです。

変数とデータを等価にする命令だから「=」を使うという説明を噂で聞きました。

値が既に代入されている変数にもう一度データを代入すると、変数に格納されているデータが更新されます。 int num; num = 10; num = 25; 上のコードでは「num」変数には「25」が格納されている状態になります。

変数の初期化

「変数の宣言」と「変数の代入」を分けて行うのは大変ですよね、、、

これを一度にまとめて行うことも可能です。
これを変数の初期化と呼びます。

以下のように書きます。 int num = 10; 一般的には変数を宣言したが、代入していない状態は危険であるため、初期化をすることをオススメします。
危険とは、何も値が代入されていない変数を参照すると例外が発生することを意味します。

データ型の推論

初期化をする場合には、最初に代入するデータによってその変数のデータ型を推論することが可能です。
これを「データ型の推論」と言い、「var」キーワードを使用して実現可能です。

以下のように書きます。 var num = 10; しかしながら、この方法では思わぬ例外が発生することがあるので注意して下さい。
例えば、数値が整数型と推論されて、小数型のデータを格納できなくなるといったことが想定されます。 「10.0」として強制的に小数型として推論させるか、「10D」のようにリテラルデータにデータ型を推論させます。 僕は「10D」のようにリテラルデータにデータ型を指定することをオススメします。
理由は、そちらのほうがコードが読みやすいからです。

変数の参照

変数に格納されているデータを使用するには、変数をそのまま書けばOK!です。 var num = 10; Console.WriteLine(num); 「Console.WriteLine」に関しては次の標準入出力で説明します。
データを出力するメソッドだと認識して下さい。

これを実行すると、「num」変数に格納されたデータをコンソール画面に表示します。

上の例では、「10」と出力されます。

標準入出力

コンソール画面にデータを表示したり、コンソール画面からデータを取得する手法について説明します。

標準出力

最初にコンソール画面にデータを出力する方法を説明します。
以下のように書きます。 Console.Write(データ); // or Console.WriteLine(データ); 「Write」は表示するデータをそのまま、「WriteLine」は最後に改行文字が自動で挿入して表示します。

標準入力

コンソール画面に入力されたデータを受け取ります。
以下のように書きます。 var 変数 = Console.ReadLine(); これで、コンソール画面に入力された文字列を変数に格納できます。 var word = Console.ReadLine(); Console.WriteLine("Hello " + word + " !!!"); コンソール画面から数字を受け取る際には、まず文字列型として受け取り、それを数値型に変換する処理をします。
データ型の変換については、後ほど説明します。
「Console.Read」は最後の一文字だけを受け取り、かつその文字のASCIIコードを返すため、ほとんど使用する機会はありません。
